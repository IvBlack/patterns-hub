Шаблон: строитель.
****************
Суть:

Отделить контруирование сложного объекта от его представления таким образом, чтобы в результате
одного и того же конструирования мы могли получить разные представления.
****************

Для чего:
Создание различных объектов из одного набора данных.
****************

Пример:
- порядок создания объекта не зависит от того, из каких частей состоит объект и как они взаимосвязаны.
- процесс конструирования объекта должен предоставлять различные представления объекта, который мы конструируем.
- например, студия по производству различных сайтов.

Шаблон: адаптер.
****************
Суть:

Преобразование интерфейса одного типа в интерфейс другого типа.
****************

Для чего:
Обеспечение совместной работы классов, интерфейсы которых несовместимы.
****************

Пример:
- интерфейс класса, который мы хотим использовать не соответствует нашим потребностям.
- необходим класс, который должен взаимодействовать с классами, еоторые ему неизвестны или
не связаны с ним.
- необходимо использовать несколько существующих подклассов, но нецелесообразно использовать методы этих классов,
создавая их новые подклассы.

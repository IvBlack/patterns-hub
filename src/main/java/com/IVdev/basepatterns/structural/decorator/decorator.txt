Шаблон: декоратор.
****************
Суть:
динамическое добавление новых обязанностей объекту.
****************

Для чего:
для расширения функциональности
используется в качестве альтернативы порождению подклассов.
****************

Пример:
- расширение классов путем порождения подклассов невозможно по какой-либо причине.
- реализация обязанностей, которые могут быть сняты с объекта.
